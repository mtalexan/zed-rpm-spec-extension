# RPM Spec Language Support for Zed

Language support for [RPM `.spec` files](https://rpm-software-management.github.io/rpm/manual/spec.html) in the [Zed editor](https://zed.dev).

## Features

- **Syntax highlighting** via the [tree-sitter-rpmspec](https://gitlab.com/cryptomilk/tree-sitter-rpmspec) grammar, including rpmbash injection for scriptlet bodies
- **Completions** — architecture value completions after `%ifarch` and `%ifnarch` directives (e.g. `x86_64`, `aarch64`, `%{arm}`)
- **Hover** — shows the defined value of `%global`/`%define` variables on hover; shows descriptions for preamble tags (`Name`, `Version`, `BuildRequires`, etc.)
- **Go to definition** — jump to the `%global` or `%define` declaration for a macro variable
- **Find references** — locate all `%{varName}` usages and `%undefine` calls for a variable
- **Snippets** — templates for full spec files, sub-packages, conditionals, and individual sections
- **Bracket matching** — for `%{…}`, `%(…)`, `${…}`, `[…]`, `(…)`, and quotes

## Installation

Search for **RPM Spec** in the Zed Extensions panel (`zed: extensions`).

## Grammar

Syntax highlighting and language injection use the [`tree-sitter-rpmspec`](https://gitlab.com/cryptomilk/tree-sitter-rpmspec) grammar (MIT License). This grammar provides:

- Full parsing of RPM spec file syntax
- `rpmbash` — an extended bash grammar for scriptlet sections that understands RPM macros

## Language Server

The `rpm-spec-ls` language server provides completions, hover, definition, and reference support. It is compiled as a standalone Rust binary.

On first use, Zed will attempt to find `rpm-spec-ls` on your `PATH`. If it is not found, the extension will download a pre-built binary for your platform from the GitHub Releases page.

Supported platforms for automatic download:

- Linux x86\_64
- macOS x86\_64
- macOS aarch64 (Apple Silicon)

To build and install the server manually:

```sh
cargo build --release --manifest-path rpm-spec-ls/Cargo.toml
cp rpm-spec-ls/target/release/rpm-spec-ls ~/.local/bin/
```

## Development

To test the extension locally, install it as a dev extension in Zed:

1. Open Zed
2. Open the Command Palette and run `zed: install dev extension`
3. Select the directory containing this repository

> **Warning:** Zed clones grammar repositories via HTTPS into the `grammars/` directory during a dev extension rebuild. If a `grammars/` directory already exists and was cloned over SSH (e.g. because your global Git config sets `url."git@github.com:".insteadOf = "https://github.com/"` or similar), Zed will error out. If you have SSH-rewriting configured, you must manually remove the `grammars/` folder before each rebuild:
> ```sh
> rm -rf grammars/
> ```

If your changes **do not modify `rpm-spec-ls`**, no extra steps are needed — Zed will use whatever binary is already on your `PATH` or download a pre-built one.

If your changes **include modifications to `rpm-spec-ls`**, you must build and install the binary locally before testing, otherwise Zed will use the old version. Because the extension checks `PATH` before attempting a download, placing the locally built binary on your `PATH` is sufficient:

```sh
cargo build --release --manifest-path rpm-spec-ls/Cargo.toml
cp rpm-spec-ls/target/release/rpm-spec-ls ~/.local/bin/
```

This means you do **not** need to push changes to GitHub or create a release to test the extension end-to-end.

## Releasing

Pushing a tag of the form `v*` triggers the GitHub Actions workflow that builds `rpm-spec-ls` for all supported platforms and attaches the binaries to a GitHub Release.
